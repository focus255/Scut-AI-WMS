## 一、出库单创建（拆零拣选 + 重新封装）

### 核心流程

出库单创建时，系统立即执行拆零拣选和重新封装。

```
用户创建出库单
    │  明细: materialCode=MAT001, packCapacity=15, planQty=90
    │
    ▼
FIFO 拆零拣选
    │  查询 type='inbound' AND status='在库' AND remainingQty>0 的条码
    │  三级排序: inboundOrder.createdAt → barcode.createdAt → barcode.id
    │
    ├── 入库箱1 (20件,30天前) ──拆空→ remainingQty=0, status='已出库'
    ├── 入库箱2 (20件,28天前) ──拆空→ remainingQty=0, status='已出库'
    ├── 入库箱3 (20件,26天前) ──拆空→ remainingQty=0, status='已出库'
    ├── 入库箱4 (20件,24天前) ──拆空→ remainingQty=0, status='已出库'
    └── 入库箱5 (20件,22天前) ──拆10→ remainingQty=10, status='在库'（FIFO队头）
        已拣: 90件 ✓
    │
    ▼
重新封装
    │  90件 ÷ 15件/箱 = 6 个出库标签
    │
    ├── OUT|MAT001|CK20260615...|15|90|15|1  status='待出库'
    ├── OUT|MAT001|CK20260615...|15|90|15|2  status='待出库'
    ├── ...
    └── OUT|MAT001|CK20260615...|15|90|15|6  status='待出库'
    │
    ▼
扣减库存
    └── inventory.stockQty -= 90
```

### 调用链条

```
前端 InboundOutbound.vue
  │
  ├── [出库Tab] 新建出库单按钮 @click="openOutboundDialog"
  ├── openOutboundDialog() → 打开对话框，物料远程搜索
  │
  ├── 保存按钮 @click="handleOutCreate"
  │     └── createOutbound({ details })
  │
  ▼
前端 API
  └── [outbound.js:13] POST /api/outbound/orders
  │
  ▼
后端 OutboundServiceImpl.create() @Transactional
  │
  ├── 生成单号: CK + yyyyMMddHHmmssSSS
  ├── 插入 outbound_orders (status='未出库')
  │
  └── 逐行 createDetailAndPick():
        ├── 插入 outbound_details
        ├── 校验库存 (inventory.stockQty >= planQty)
        ├── FIFO 查询入库条码 (type='inbound', status='在库')
        ├── 逐箱拆出 → 更新入库条码 remainingQty + status
        ├── 封装出库标签 → 插入 barcodes (type='outbound', status='待出库')
        └── 扣减库存 inventory.stockQty -= planQty
```

### 出库标签格式

```
OUT | MAT001 | CK20260615185611594 | 15 | 90 | 15 | 1
 │       │              │            │     │    │    │
 │       │              │            │     │    │    └── 箱号
 │       │              │            │     │    └── 本箱数量
 │       │              │            │     └── 计划总数
 │       │              │            └── 出库箱容量
 │       │              └── 出库单号
 │       └── 物料编码
 └── 协议头（区别于入库条码的 WMS 前缀）
```

## 二、FIFO 先进先出

### 三级排序规则

```
第1级: InboundOrder.createdAt  → 入库单创建时间（跨批次排序）
第2级: Barcode.createdAt       → 条码创建时间（批次内箱间排序）
第3级: Barcode.id              → 自增ID（兜底）
```

### 时间错开策略

| 场景 | 策略 |
|------|------|
| 种子数据 | 每物料错开 2 天，同物料内箱号逐毫秒递增 |
| 正式入库 | 批量生成条码后 `plusSeconds(i)` 逐箱错开 1 秒 |
| 拆箱出库 | 最后一箱有余量时保留 `status='在库'`，作为 FIFO 队头 |

### 校验位置

| 方法 | 位置 | 说明 |
|------|------|------|
| `create()` 拆零拣选 | `OutboundServiceImpl.java` | 创建时自动按 FIFO 排序拆箱 |
| `validateFifo()` | `OutboundServiceImpl.java:301` | 手动确认出库时强校验条码顺序 |
| `scanOutbound()` | `OutboundServiceImpl.java` | 扫码出库直接核销已封装的出库标签 |

## 三、确认出库

### 方式一：手动确认

```
前端 InboundOutbound.vue → 出库Tab → 确认出库按钮
  │
  ├── openOutConfirmDialog(row)
  │     └── 加载出库单详情（含出库标签列表）
  │
  ├── 每行输入本次出库数 + 扫描条码（tag输入，支持扫码枪）
  │
  └── handleOutConfirmSubmit()
        └── confirmOutbound(id, { details: [{ detailId, actualQty, barcodes }] })
  │
  ▼
后端 OutboundServiceImpl.confirm() @Transactional
  │
  ├── 校验条码: status='在库' AND 物料匹配
  ├── validateFifo(): 用户输入的条码顺序必须等于 FIFO 顺序
  ├── 数量校验: 本次 = 条码折算总数量
  ├── 累计校验: 已出 + 本次 ≤ 计划
  ├── 库存校验: stockQty ≥ confirmedQty
  ├── 写入 outbound_histories（每条码一条流水）
  ├── 更新 barcode.status → '已出库'
  ├── 更新 outbound_detail.actualQty
  └── 更新 outbound_order.status（未出库/部分出库/已完成）
```

### 方式二：扫码出库

```
Dashboard / 任意页面 → 扫描 OUT|... 标签
  │
  ▼
统一扫码入口 POST /api/outbound/scan
  │
  ├── 判定: barcode.startsWith("OUT|") → 出库
  │
  ▼
OutboundServiceImpl.scanOutbound()
  │
  ├── 查找出库标签: type='outbound' AND barcode=?
  ├── 校验状态: '待出库' / '已出库' → 拒绝重复扫码
  ├── 标记: status='已出库'
  ├── 更新 outbound_detail.actualQty += boxQty
  ├── 记录 outbound_histories（来源标注"拆零重封装"）
  └── 更新 outbound_order.status
```

## 四、出库历史 / 流水

### 数据表

```
outbound_histories
  ├── id (PK)
  ├── outbound_id → outbound_orders.id
  ├── outbound_order_no
  ├── outbound_detail_id
  ├── material_code
  ├── inbound_id         (入库条码为实际入库单ID，出库标签为0)
  ├── inbound_order_no   (入库条码为实际单号，出库标签为"拆零重封装")
  ├── barcode_id
  ├── barcode            (出库时扫的条码号)
  ├── deduct_qty         (本次扣减数量)
  └── created_at
```

### 查询入口

| 入口 | 位置 |
|------|------|
| 出库单详情 → 出库流水表 | `InboundOutbound.vue` 出库详情对话框 |
| 出库流水独立查询 | `InboundOutbound.vue` 出库流水查询对话框 → `GET /api/outbound/histories` |
| 库存追溯 | `InventoryTrace.vue` → 显示 outboundOrderNo + outboundAt |

## 六、出库单编辑 / 删除

### 编辑流程

```
PUT /api/outbound/orders/{id}
  │
  ├── 校验: status ≠ '已完成'
  ├── rollbackPick(): 退回已拣库存
  │     ├── 统计出库标签总数量
  │     ├── 恢复入库条码 remainingQty + status
  │     └── 恢复 inventory.stockQty
  ├── 删除旧出库标签 (type='outbound')
  ├── 删除旧明细 + 旧流水
  ├── 重新 createDetailAndPick() → 拆零拣选 + 封装
  └── 重置 status='未出库'
```

### 删除流程

```
DELETE /api/outbound/orders/{id}
  │
  ├── 校验: status = '未出库'
  ├── rollbackPick(): 退回库存
  ├── 清理: 出库标签 + 明细 + 流水
  └── 删除 outbound_orders 记录
```

## 七、统一扫码（出入库双向）

```
POST /api/outbound/scan
  │
  ├── barcode.startsWith("OUT|") ?
  │     ├── 是 → outboundService.scanOutbound(barcode)
  │     └── 否 → inboundService.scanReceive(request)
  │
  └── 返回 ScanResponse { type, materialCode, orderNo, qty, barcode }
```

前端 Dashboard 扫码区标题"扫码出入库"，结果区自动显示入库(绿)/出库(灰)不同样式。

## 八、数据库表关系

```
outbound_orders (出库单主表)
  ├── id (PK)
  ├── order_no (唯一单号, CK开头)
  ├── status (未出库/部分出库/已完成)
  ├── created_at
  └── updated_at

outbound_details (出库明细)
  ├── id (PK)
  ├── outbound_id (FK → outbound_orders.id)
  ├── material_code
  ├── pack_capacity (出库箱容量)
  ├── plan_qty (计划出库数)
  └── actual_qty (实际出库数)

outbound_histories (出库流水)
  ├── id (PK)
  ├── outbound_id → outbound_orders.id
  ├── inbound_id → 来源单据ID
  ├── barcode_id → barcodes.id
  ├── barcode (条码号)
  └── deduct_qty (扣减数量)

barcodes (条码追踪 — 出入库共用)
  ├── id (PK)
  ├── type (inbound 入库条码 / outbound 出库标签)
  ├── barcode (WMS|... 或 OUT|... 格式)
  ├── inbound_id (入库条码→入库单ID / 出库标签→出库单ID)
  ├── status (入库:待入库/在库/已出库; 出库:待出库/已出库)
  ├── remaining_qty (剩余数量，拆箱后更新)
  ├── created_at
  └── updated_at
```
